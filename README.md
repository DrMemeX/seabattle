# Sea Battle

Консольная игра «Морской бой» на Java с модульной архитектурой, режимом игры против бота, ручной и автоматической расстановкой кораблей, WebSocket-MVP для игры двух игроков и текстовым отчётом по завершённой партии.

## Стек

* Java 17
* Maven
* JUnit 5
* Spring Boot WebSocket

## Структура проекта

`sea-battle-core` содержит основную игровую логику: поле, корабли, выстрелы, правила расстановки, ботов, парсеры, отчёты и тесты.

`sea-battle-console` содержит консольную реализацию игры: ввод игрока, вывод полей, ручную расстановку, запуск партии.

`sea-battle-websocket` содержит WebSocket-MVP для игры двух игроков через подключение к серверу.

## Правила игры

Игровое поле имеет размер `16x16`.

Координаты вводятся в формате:

```text
A1
D2
P16
```

Флот состоит из кораблей:

```text
6 клеток — 1 корабль
5 клеток — 2 корабля
4 клетки — 3 корабля
3 клетки — 4 корабля
2 клетки — 5 кораблей
1 клетка — 6 кораблей
```

Корабли не могут пересекаться или соприкасаться по горизонтали, вертикали и диагонали.

Результаты выстрела:

```text
MISS — мимо
HIT — ранил
KILL — убил
```

При промахе ход переходит противнику. При попадании игрок ходит повторно.

## Запуск консольной игры

Из корня проекта:

```bash
mvn clean install
```

Затем запустить `Application`:

```text
ru.senla.seabattle.console.app.Application
```

Или через скрипт:

```bat
run-console.bat
```

В консольном режиме игра предлагает:

```text
1. Ввести имя игрока
2. Выбрать сложность бота
3. Выбрать ручную или случайную расстановку кораблей
4. Играть через ввод координат
```

Доступные уровни бота:

```text
1 — лёгкий
2 — нормальный
3 — сложный
4 — эксперт
```

## WebSocket-режим

Запуск сервера:

```bat
run-websocket.bat
```

Или вручную:

```bash
mvn -pl sea-battle-websocket spring-boot:run
```

Адрес подключения:

```text
ws://localhost:8080/ws/game
```

### Создание игры

Первый игрок отправляет:

```json
{
  "type": "CREATE_GAME",
  "name": "Андрей"
}
```

Сервер возвращает `gameId` и `playerId`.

### Подключение второго игрока

Второй игрок подключается к тому же WebSocket-адресу и отправляет:

```json
{
  "type": "JOIN_GAME",
  "gameId": "GAME_ID_ИЗ_ОТВЕТА",
  "name": "Миша"
}
```

После подключения второго игрока игра начинается.

### Выстрел

Выстрел отправляется в формате:

```json
{
  "type": "SHOT",
  "coordinate": "D2"
}
```

Стрелять может только игрок, чей `playerId` совпадает с `currentPlayerId`.

## Отчёты партий

После консольной игры создаётся текстовый отчёт в папке:

```text
reports/
```

Отчёт содержит:

```text
- время начала игры
- имена игроков
- список ходов с временем
- координаты и результаты выстрелов
- время окончания игры
- победителя
- статистику игроков
- финальные поля игроков
- поля противника с отметками попаданий
```

## Тесты

Запуск всех тестов:

```bash
mvn clean install
```

На текущий момент покрыты:

```text
- парсеры координат и ориентации
- игровое поле
- отрисовка поля
- выстрелы
- игровой движок
- правила расстановки кораблей
- случайная расстановка флота
- стратегии ботов
- запись отчёта игры
- WebSocket-сессии
- сервис WebSocket-игры
```

## Реализовано

```text
- консольная игра против бота
- выбор сложности бота
- ручная расстановка кораблей
- автоматическая расстановка кораблей
- проверка корректности расстановки
- обработка MISS / HIT / KILL
- повтор хода при попадании
- текстовый отчёт партии
- WebSocket-MVP для игры двух игроков
- тестовое покрытие основной логики
```

Главный сценарий является играбельным: можно запустить консольную партию против бота, расставить корабли, играть до победы и получить текстовый отчёт по завершённой игре.
